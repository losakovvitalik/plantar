# CLI

Запуск: `cd apps/cli && pnpm start <команда> [опции]`

## Общие опции подключения

Есть у всех команд:

| Опция | Описание |
| --- | --- |
| `--host <host>` | адрес сервера (обязательно) |
| `--user <user>` | имя пользователя (обязательно) |
| `--port <port>` | SSH-порт, по умолчанию `22` |
| `--key <path>` | путь к приватному ключу |
| `--password <pass>` | пароль (если без ключа) |

Нужно указать либо `--key`, либо `--password`.

## Команды

### `ls`

Список директорий на сервере.

- `--path <path>` — какую директорию смотреть, по умолчанию `.`

### `info`

ОС, ресурсы (CPU, RAM, диск) и установленные инструменты (node, pnpm, pm2, nginx, certbot). Показывает, поддерживается ли сервер (Ubuntu 22.04/24.04).

### `setup`

Устанавливает недостающие инструменты: Node.js 22 (NodeSource), pnpm, pm2, nginx, certbot. Уже установленные пропускает — можно запускать повторно.

### `deploy`

Собирает проект локально и деплоит на сервер: загрузка в `/var/www/<name>`, настройка nginx, выпуск SSL-сертификата (если указан `domain`). Идемпотентна.

- `--project <dir>` — папка проекта с `plantar.json`, по умолчанию текущая

### `logs`

Логи nginx по сайту (access и error).

- `--project <dir>` — папка проекта с `plantar.json`, по умолчанию текущая
- `--lines <n>` — сколько последних строк показать, по умолчанию `50`

## plantar.json

Лежит в корне деплоимого проекта:

```json
{
  "name": "my-app",
  "buildCommand": "pnpm build",
  "buildDir": "dist",
  "domain": "example.com"
}
```

| Поле | Описание |
| --- | --- |
| `name` | имя сайта (обязательно): строчные латинские буквы, цифры, дефис; определяет `/var/www/<name>` |
| `buildCommand` | команда сборки, по умолчанию `npm run build` |
| `buildDir` | папка со сборкой, по умолчанию `dist` |
| `domain` | домен сайта; если не указан — сайт отвечает по IP сервера, без HTTPS |

Пример: [examples/react-demo-app](../examples/react-demo-app/plantar.json).
