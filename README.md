# Plantar

Десктопное приложение для быстрого деплоя React, Next.js и Node.js-приложений на Ubuntu-серверах.

Ориентировано в первую очередь на пользователей не-программистов. Пользователь с любым уровнем владения ПК должен иметь возможность задеплоить приложение без ручного подключения к серверу и выполнения команд.

## Установка

Готовые сборки можно скачать на странице [последнего релиза](https://github.com/losakovvitalik/plantar/releases/latest):

- **macOS** — файл `.dmg` (`arm64` для компьютеров с чипом Apple, `x64` для Intel).
- **Windows** — файл `.exe`.
- **Linux** — файл `.AppImage`.

Сборки пока не подписаны сертификатами, поэтому система при первом запуске предупредит о неизвестном разработчике:

- **macOS**: если приложение не открывается, зайдите в «Системные настройки» → «Конфиденциальность и безопасность», найдите внизу сообщение о заблокированном приложении Plantar и нажмите «Открыть». Это нужно сделать один раз.
- **Windows**: в окне «Система Windows защитила ваш компьютер» нажмите «Подробнее», затем «Выполнить в любом случае».

## Главный сценарий

1. Подключиться к серверу по SSH.
2. Подготовить сервер: проверить ОС, ресурсы, доступы и зависимости.
3. Задеплоить React, Next.js, Node.js-приложение или Telegram-бота.
4. Показать логи. Процесс деплоя должен быть прозрачным.
5. Повторно запуститься без поломки. Важна идемпотентность.
6. Понятно упасть с ошибкой и показать, что пошло не так.

## Принципы

- Local-first: все операции выполняются на компьютере пользователя.
- Без внешнего backend-сервиса для деплоя.
- SSH-доступы и env-переменные не должны покидать устройство пользователя. Единственное осознанное исключение — «Деплой при коммите»: для него создаётся отдельный deploy-ключ (не личный ключ пользователя), и этот ключ вместе с адресом сервера сохраняется в GitHub Secrets — иначе GitHub Actions не смог бы деплоить без участия приложения.
- Пользователь должен видеть, какие действия выполняются на сервере.
- GUI не должен содержать deploy-логику, а только вызывать общий core.

## MVP

Первая версия поддерживает только:

- Ubuntu 22.04 / 24.04.
- React-приложения.
- Node.js.
- pnpm.
- pm2.
- nginx.
- Let's Encrypt.

## Архитектура

Две независимые части: CLI (`apps/cli`) и GUI (`apps/desktop`).

Это упрощает разработку и тестирование. CLI можно будет использовать отдельно, если кто-то не захочет использовать UI. Но в первую очередь задача CLI — обслуживать десктопное приложение.

Общая deploy-логика должна жить в отдельных пакетах:

```text
packages/
  core/
  ssh/
  config/
  recipes/
  types/
```

## Roadmap

- Подключение к серверу по SSH.
- Проверка ОС и базовой информации о сервере.
- Установка Node.js, pnpm, pm2, nginx и certbot.
- Деплой React-приложения.
- Настройка nginx.
- Выпуск SSL-сертификата.
- Просмотр логов.
- Повторный деплой без поломки существующей конфигурации.
- Локальное хранилище: конфиг проекта (`plantar.json`), история деплоев, логи.
- Хранилище серверов и ключей: секреты шифруются через системный keychain (`safeStorage`), не хранятся открытым текстом.
- Desktop GUI.

## Статус

Проект находится на ранней стадии разработки. Пока не стоит использовать его на production-серверах без ручной проверки выполняемых команд.
